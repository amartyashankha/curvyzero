# CurvyTron Raw Run Probe

Date: 2026-05-08

## Short Answer

We cannot run the raw CurvyTron reference clone right now.

The documented run command is:

```sh
node bin/curvytron.js
```

That file does not exist in this clone yet. It is generated by Gulp. The generated browser files are also missing, including `web/index.html`, `web/js/curvytron.js`, `web/js/dependencies.js`, and `web/css/style.css`.

The smallest safe next experiment is to install and build in a disposable copy of the reference clone, not in `third_party/curvytron-reference`. Use an old Node version if possible, because this project uses Gulp 3, Bower, and very old `gulp-sass` / `node-sass`.

## Package Setup

`package.json`:

- Project version: `curvytron@1.4.5`.
- Runtime dependencies: `express`, `faye-websocket`, `influx`.
- Optional dependencies: `usage`, `MD5`.
- Build dependencies: `gulp@^3.8.*` and old Gulp plugins.
- Install script: `bower install`.

`bower.json`:

- Client dependencies include Angular 1.4.3, Angular route/cookies, SoundJS, color picker, and several `tom32i-*` packages.

Docs say:

```sh
npm install
bower install
gulp
node bin/curvytron.js
```

One wrinkle: `bower` is not listed in `package.json`, and it is not installed globally on this machine.

## Build Setup

`gulpfile.js` uses Gulp 3 task syntax.

Important tasks:

- `server`: concatenates server files into `bin/curvytron.js`.
- `front-expose`: concatenates Bower browser dependencies into `web/js/dependencies.js`.
- `front-min` / `front-full`: build client app JS into `web/js/curvytron.js`.
- `ga`: builds `web/index.html` from `src/client/views/index.html`.
- `views`: copies/minifies Angular views into `web/js/views`.
- `sass-min` / `sass-full`: builds `web/css/style.css`.
- `default`: runs lint, server build, client dependency build, HTML/views, minified client JS, and minified CSS.
- `dev`: same idea, but unminified and includes `stressTest.js`.

## Entry Points

The real server entry point is generated:

```sh
node bin/curvytron.js
```

`src/server/launcher.js` is not a standalone Node entry point. It expects to be concatenated into `bin/curvytron.js`; when run directly, its relative `require('../package.json')` points at the wrong place.

The server uses Express static files from `web`:

```js
this.app.use(express['static']('web'));
```

So the browser game needs both the generated server bundle and the generated web files.

## Services Needed

Default local play appears to need only Node.

No Mongo or Redis references were found. `influx` exists for the optional inspector only. The sample config has:

```json
"inspector": {
  "enabled": false
}
```

If the inspector is enabled, it expects InfluxDB. With the default or missing config, the inspector is off.

## Blockers

Hard blockers right now:

- Dependencies are not installed.
- `bower` is not available globally.
- The generated server file `bin/curvytron.js` is missing.
- The generated browser files under `web` are missing.

Likely blocker after install:

- The build chain is from the Node 0.10 era. Gulp 3 and `gulp-sass@0.7.3` / `node-sass@0.9.6` are very likely to break on Node 25. The npm dry-run can resolve package names, but that does not prove the native Sass dependency can install or run.

## Least Invasive Run Path

In a disposable copy, the raw path to try is:

```sh
npm install
bower install
gulp
node bin/curvytron.js
```

Then open:

```text
http://localhost:8080/
```

But I would not run that inside `third_party/curvytron-reference` as the next step. It would write many dependency and build files into the reference clone.
