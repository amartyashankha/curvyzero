# Decision Records

Decision records capture choices that shape implementation. They should be short, dated, and reversible where possible.

## Records

- `0001-investigation-first-repo-structure.md`
- `0002-modal-hot-loop-locality.md`
- `0003-multiplayer-selfplay-v0-formulation.md`

- `0005-main-pong-repository-library-choice.md`

## Status Values

- Proposed - plausible, not yet binding.
- Accepted - current project direction.
- Superseded - replaced by a newer decision.
- Rejected - considered and intentionally not chosen.

## Template

```md
# ADR-XXXX: Title

Date: YYYY-MM-DD
Status: Proposed

## Context

## Decision

## Evidence

## Consequences

## Reversal Conditions

## Links
```
