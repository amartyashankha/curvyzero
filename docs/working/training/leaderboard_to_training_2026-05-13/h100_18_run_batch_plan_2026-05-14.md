# H100 18-Run Batch Plan

Date: 2026-05-14

Scope: docs-only plan for tonight's H100 training batch. Do not launch from this
doc until the leaderboard gate is satisfied.

2026-05-15 supersession: do not use this as launch guidance. The fixed base
below uses H100 compute with CPU `body_circles_fast + simple_symbols`, which is
historical/control evidence only. Current production policy observations are
CPU `cpu_oracle` `browser_lines + simple_symbols`; GPU
`browser_lines + simple_symbols` remains lab/profiling-only until
trainer-visible contract parity passes.

## Hard Gate

Leaderboard-derived opponents must come from a non-provisional final tournament
snapshot. Before launch, resolve each rank reference to an exact immutable
checkpoint ref, such as `.../ckpt/iteration_N.pth.tar`, and record the source
snapshot id/hash in the manifest audit.

Do not use provisional rank rows, live leaderboard pointers, rank labels without
checkpoint refs, or `latest` aliases as launch inputs.

## Naming Scheme

Use:

```text
h100_<reward>_<mix>_<stochasticity>_<snapshot_tag>
```

Example:

```text
h100_surv_bonus_s2_stoch10_finaltop4_20260514
```

The `snapshot_tag` should identify the final tournament source, not a live
leaderboard pointer.

## Launch Blockers

- Final non-provisional tournament snapshot is not identified and hashed.
- Rank 1-4 rows are not resolved to exact immutable checkpoint refs.
- `stoch10` support is not verified for both requested knobs. If repeat/skip-like
  behavior is unsupported, either remove that half from `stoch10` or rename the
  mode to match the implemented behavior.
- Reward profile names are not mapped to trainer-supported flags or recipes.
- Blank opponent entry is not confirmed to use the trainer-supported blank or
  scripted no-op mechanism.
- Tonight manifest includes a 5% immortal wall-avoidant scripted slot in every
  mix; frozen leaderboard checkpoint slots should remain mortal/normal.
- Do not seed the learner weights from rank1 for tonight. There is no clean
  model-only leaderboard init API yet; the existing resume path is same-run
  resume and can carry incompatible learner state across reward variants.
- Manifest/audit paths are not written before launch.
