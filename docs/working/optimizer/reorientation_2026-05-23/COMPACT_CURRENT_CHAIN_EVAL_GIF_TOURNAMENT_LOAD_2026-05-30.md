# Compact Current-Chain Eval/GIF/Tournament Load

Date: 2026-05-30

Status: local contract landed; real local current-chain smoke passed.

## What Landed

Added
`curvyzero_compact_current_chain_eval_gif_tournament_load_evidence/v1` in
`src/curvyzero/training/compact_eval_gif_tournament_load.py`.

The contract is a sibling evidence object. It does not mutate the compact
checkpoint or the stock-shaped export. It validates and hashes:

- compact trainer checkpoint,
- derived stock export,
- policy-observation sidecar,
- strict stock model-load verifier report,
- tournament-loader report,
- stock-export evidence bundle,
- one-game/GIF gameplay smoke report,
- game summary,
- `game.gif`,
- `frames.npz`,
- standalone visual-survival eval summary.

The evidence flips only the local evidence claim
`eval_gif_tournament_load=true`. It keeps `promotion_claim=false`,
`training_speed_claim=false`, `calls_train_muzero=false`, and
`touches_live_runs=false`.

## Honesty Tightening

Strict model load is no longer treated as tournament/gameplay loadability.
`verify_compact_stock_export_model_contract_v1(...)` can report
`stock_model_contract_verified=true` and
`strict_stock_model_load_verified=true`, but it leaves
`stock_eval_tournament_loadable=false` until gameplay/GIF/eval evidence exists.

The stock-export evidence bundle now stays loader-scoped. It can bind strict
load and tournament-loader construction by file hash, but
`stock_eval_tournament_loadable_by_evidence=false` and
`eval_gif_tournament_loadable_by_evidence=false` until the separate current-chain
contract validates a gameplay/GIF/eval report.

Coach compatibility now rejects arbitrary evidence strings for
`eval_gif_tournament_load=true`. The evidence ref must start with:

```text
compact_current_chain_eval_gif_tournament_load:
```

This blocks loader-only strings such as `unit_loader_smoke_no_game_or_gif` from
closing the full eval/GIF/tournament gate.

## What This Does Not Prove

This is a local eval/GIF/tournament-load smoke. It does not prove Coach speed,
promotion quality, stock resume, live-run safety, or that all required
promotion gates are unified on this exact checkpoint.

Important caveat: the current durable normal-death checkpoint smoke builds a
tiny `torch.nn.Linear(3,2)` compact checkpoint. That checkpoint is useful for
death/terminal, resume, metrics, and policy-refresh contracts, but it is not a
stock LightZero-shaped model and should fail stock strict-load/GIF/eval proof
unless a separate LightZero-shaped compact checkpoint/export chain is produced.
