# MuZero Cold Start Vs Curriculum Critique - 2026-05-09

Role: cold-start sanity critic. No pytest was run.

## Short Answer

No: MuZero does not normally require a hand-built curriculum.

The original and official setups rely on repeated environment interaction,
planning with Monte Carlo tree search, replay, exploration during collection,
large enough data/update budgets, and task-correct observation/reward setup.
For board games, that means self-play or bot-play trajectories with final
outcome targets. For Atari Pong, that means the real Atari environment,
frame-stack visual input, a convolutional model, action repeat/frame handling
inside the Atari wrapper, replay, search, and scale.

So if our question is "does MuZero need a Pong curriculum to learn at all?",
the answer is no.

If our question is "is a tiny custom dummy Pong curriculum a useful diagnostic
after our sparse toy collapsed under longer runs, more updates, and simple
exploration?", the answer is yes, but only if it is tightly bounded and still
judged on normal heldout score. Otherwise it becomes a rabbit hole.

## What Official Setups Rely On Instead

Original MuZero:

- learns reward, policy, and value predictions relevant to planning;
- chooses actions through tree search over the learned model;
- trains from data generated by MuZero itself, not external demonstrations;
- uses self-play for board games and environment interaction for Atari;
- learns from stored trajectories in replay.

Official LightZero Atari Pong:

- environment: `PongNoFrameskip-v4`;
- observation: stacked grayscale frames, `(4, 64, 64)`;
- model: convolutional MuZero;
- action space: Atari Pong's 6 discrete actions;
- collection: `collector_env_num=8`, `n_episode=8`;
- search: `num_simulations=50`;
- update/replay: `update_per_collect=None`, `replay_ratio=0.25`;
- batch/scale: `batch_size=256`, `max_env_step=500000`;
- segment variant: `num_segments=8`, `game_segment_length=20`,
  `train_start_after_envsteps=2000`.

Official LightZero sparse board-game bot configs:

- TicTacToe bot mode: `collector_env_num=8`, `n_episode=8`,
  `num_simulations=25`, `update_per_collect=50`, `batch_size=256`,
  `max_env_step=200000`;
- Connect4 bot mode: `collector_env_num=8`, `n_episode=8`,
  `num_simulations=50`, `update_per_collect=50`, `batch_size=256`,
  `max_env_step=500000`;
- both use `env_type=board_games`, changing legal-action masks,
  final-outcome style targets, large `td_steps`, and `discount_factor=1`.

Our custom dummy Pong sparse lane is much smaller and different:

- `tabular_ego`: 10 float features, not visual Atari frames;
- action space: 3 actions, up/stay/down;
- model: MLP, not the official Atari conv stack;
- opponent is hidden inside the env step;
- reward is true sparse score: ego scores `+1`, ego loses `-1`, otherwise `0`;
- latest sparse probes used fixed 120-step horizon, `td_steps=120`,
  `discount_factor=1`, support ranges, small collectors, and tiny budgets
  compared with official examples.

## What The Sparse Failures Say

The latest sparse failures make "just train the same thing longer" a weak
answer:

- Sparse settings ran mechanically with `td_steps=120` and
  `discount_factor=1`, but heldout MCTS still had weak/tied roots and no
  reliable controller.
- Rung 0 and pure 2x rung 1 did not improve heldout survival, shaped telemetry,
  raw score, or action entropy. Final/best checkpoints collapsed all-up.
- Higher update/replay (`update_per_collect=25`, `batch_size=64`) did not fix
  heldout quality. `iteration_50` was still mostly up, and `ckpt_best` was
  all-up.
- Random warmup plus epsilon collection diversified train actions, but heldout
  eval still collapsed and raw score stayed bad versus `lagged_track_ball_1`
  and `random_uniform`.
- The first `contact_pressure` curriculum smoke passed mechanically. It is an
  opt-in reset profile; the default reset stayed unchanged; sparse reward was
  preserved as `+1/-1/0`. The tiny train was non-flat, but heldout quality
  failed and eval still used zero `down`.

This does not prove a curriculum is the answer. It proves the old sparse toy
setup has not earned more same-shape scale.

## Curriculum: Diagnostic Or Rabbit Hole?

A custom dummy Pong curriculum is a legitimate diagnostic if it answers one
narrow question:

Can MuZero learn from score-bearing Pong states when those states are common
enough in replay, while the reward and heldout task stay honest?

For it to be worth doing, these must be true:

- `env.step()` reward remains sparse `+1/-1/0`.
- The curriculum changes reset/opponent distribution, not the definition of
  winning.
- Candidate states are admitted because true sparse rollouts show that legal
  ego actions change score outcome, not merely because contact happens.
- At least a small scoreability probe passes before training, for example
  at least 10 of 64 sampled states have nonzero true score-return spread and
  best actions are not all the same.
- Canonical normal-reset heldout eval remains separate and mandatory.
- The run compares initialization, final, and best checkpoints under the same
  scorecard.
- Pass requires raw score or win-rate movement against scoreable opponents,
  non-collapsed action histograms, and no canonical regression.

It is a rabbit hole if:

- we start saying MuZero "needs curriculum" in general;
- we use curriculum results as stock Atari Pong evidence;
- contact count, survival, or shaped loss-delay replaces raw score as the main
  claim;
- biased starts make a checkpoint look good while normal heldout gets worse;
- we keep running larger curriculum jobs after action collapse persists;
- the pre-training state probe cannot show score-return spread.

## Quick Falsifiers

Stop the curriculum path quickly if any of these happen:

- scoreability probe finds `score_delta_return_differs_state_count=0`, or fewer
  than 10/64 useful states;
- admitted states are mostly unavoidable wins, unavoidable losses, or one
  best action;
- final or best checkpoint does not beat initialization on canonical heldout
  raw score against `lagged_track_ball_1` or `random_uniform`;
- action entropy stays collapsed, or one action exceeds 90% without
  state-dependent root evidence;
- curriculum heldout improves but normal heldout regresses;
- improvement exists only in trainer-side telemetry, not independent MCTS
  scorecards;
- a second seed does not reproduce the same qualitative result.

The existing tiny `contact_pressure` smoke is already a warning: it proved the
opt-in knob, default-safety, and reward integrity, not learning quality. It is
a diagnostic knob, not an answer and not proof. It justifies at most one
modest, better-instrumented diagnostic rung, not a broad curriculum campaign.

2026-05-09 scoreability update:
`docs/experiments/2026-05-09-dummy-pong-contact-pressure-scoreability-probe.md`
ran the pre-training state probe over real `contact_pressure` resets with true
sparse reward. It passed the action-sensitivity bar overall: 192/192
reset/opponent groups changed contact angle, score return, terminal outcome, or
survival when sweeping legal ego actions, and 188/192 had score-return spread.
It did not make `track_ball` scoreable: `track_ball` was 0/64 scoreable groups,
while `lagged_track_ball_1` was 46/64 and `stay` was 59/64. The curriculum
path is therefore still narrow: go only against scoreable lagged/simple
opponent slices, and stop any claim that contact-pressure solves the hard
`track_ball` target.

## Plain Decision

We did not discover that MuZero requires curriculum. We discovered that our
tiny custom sparse dummy Pong setup is not producing usable heldout learning.

The curriculum idea is valid only as a bounded scoreability diagnostic for the
custom toy. The main project should keep official visual Pong, raster/debug,
and real self-play/environment work separate. Do not let a custom curriculum
become the main story unless it passes normal heldout scorecards quickly.
