## Claim

The currently relevant normal Pong runs are Wave11 seeds `71`, `73`, `74`, and
`76`. Seeds `73`, `74`, and `76` show strong late survival curves. Seed `71`
turns positive late. Survival-shaped seed `81` is useful side telemetry, but it

## Eval Settings

- Eval source: Modal `curvyzero-runs` Volume artifacts.
- Local manifest source: `artifacts/local/eval-manifest-files/sXX_g*.json`.
- Summarizer:
  `uv run python scripts/summarize_lightzero_pong_eval_manifest.py --survival-curve --survival-aggregate --format tsv <manifest-glob>`.
- Eval mode: stock-only, strict no-fallback checkpoint load.
- Eval cap: `2048` steps.
- Episode cap: `2048` steps.
- Search: `50` MCTS simulations per action.
- Eval starts: `8` random starts per checkpoint in the fetched `rand8-e`
  artifacts.

## Why No New Remote Eval Was Launched

The prepared pass would be `3` runs times `8` checkpoints times `8` random
starts: `192` serious stock eval jobs. Each job strict-loads a checkpoint,
starts ALE Pong, runs up to `2048` steps with `50` MCTS simulations per step,
and writes Modal Volume artifacts. That is too much to spend when the current
`rand8-e` Modal artifacts already answer the curve question.

One next tooling fix: amortize eval setup across seeds or checkpoints in a
long-lived worker, then batch artifact writes. That would make fresh 8x8
survival curves cheap enough to launch routinely.
