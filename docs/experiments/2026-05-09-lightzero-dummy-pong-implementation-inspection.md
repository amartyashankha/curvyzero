Worker A inspected the training lane for the next real dummy Pong LightZero

Correction note: LightZero/MuZero data is generated by repeated environment
interaction, so data can later be scaled with more actors, episodes, and
steps. This dummy Pong adapter is a staged sanity wrapper, not final multiplayer
self-play: it runs learner ego versus a scripted opponent, or later a frozen
checkpoint opponent depending on config. Reports must include survival steps
and shaped loss-delay score, not just wins.

Concrete facts:

- Existing LightZero files were stock-reference smokes only:
  `lightzero_cartpole_tiny_train_smoke.py`,
  `lightzero_pong_dry_config_smoke.py`, and
  `lightzero_pong_env_smoke.py`.
- No `lightzero_dummy_pong*` implementation existed under
  `src/curvyzero/training` or `src/curvyzero/infra/modal`.
- Added the missing first slice:
  `curvyzero.training.lightzero_dummy_pong_env`,
  `curvyzero.infra.modal.lightzero_dummy_pong_config_import_smoke`, and
  `curvyzero.infra.modal.lightzero_dummy_pong_tiny_train_smoke`.

The new adapter is still an ego-vs-scripted wrapper, not full simultaneous
self-play. It keeps the environment reward honest and records Pong telemetry:
wins/losses/timeouts, survival steps, score return, shaped loss-delay return,
action histograms, seeds, and trace hashes.

Known blocker after this patch: independent CurvyZero evaluation of a LightZero
`.pth.tar` checkpoint is not implemented yet. The tiny train smoke records env
sidecar rows and mirrors checkpoint files, but loading a LightZero checkpoint
outside LightZero's evaluator remains next work.
