# Design Docs

Design docs describe the current intended system. Unlike research notes, they should be coherent enough to implement from.

Current design docs:

- [deterministic_environment.md](deterministic_environment.md) - simulator scope, API, rule provenance, and early golden tests.
- [modal_architecture.md](modal_architecture.md) - Modal job architecture and reproducibility shape.
- [repository_hierarchy.md](repository_hierarchy.md) - docs/code hierarchy, stability layers, and future `src/curvyzero` package domains.
- [rulesets.md](rulesets.md) - named game rule variants and provenance expectations.
- [training_architecture.md](training_architecture.md) - baseline-first training sequence and MuZero design gates.

Design pages should stay more stable than research notes. When a design claim depends on unfinished evidence, link to the supporting research or experiment instead of embedding the whole argument here.
